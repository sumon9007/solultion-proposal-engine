# Prompt: Create Statement of Work Scope

Use this prompt when you need to generate or refine the Scope of Work section for a proposal.

---

## Purpose

The Scope of Work (SOW) is the most important section for managing client expectations and protecting commercial boundaries. It must be precise, unambiguous, and exhaustive.

---

## Instructions

Write a Scope of Work section using the following structure. Adapt based on the proposal type (project vs. managed services vs. consultancy).

---

## SOW Structure

### Introduction

One paragraph summarising what this engagement delivers and how work will be organised (phases, workstreams, or service catalogue).

---

### For Project-Based Engagements (Cloud / Implementation)

Structure work into phases. For each phase:

```markdown
#### Phase [N]: [Phase Name]

**Objective:** [One sentence — what this phase achieves]

**Activities:**
- [Specific activity 1]
- [Specific activity 2]
- [Specific activity 3]

**Our Responsibilities:**
- [What we will do]

**Client Responsibilities:**
- [What the client must do to enable this phase]

**Estimated Duration:** [X weeks / X days]
**Estimated Effort:** [X person-days]
```

Minimum 2 phases. Typical engagements have 3–5.

---

### For Managed Services Engagements

Structure as a service catalogue. For each service:

```markdown
#### [Service Name]

**Description:** [What this service provides day-to-day]

**Included Activities:**
- [Routine activity 1]
- [Routine activity 2]

**Service Level:**
- Response time: [e.g. 4 hours for Priority 1]
- Resolution target: [e.g. 8 hours for Priority 1]
- Availability: [e.g. 24x7 / Business hours]

**Reporting:** [e.g. Monthly service report, quarterly review]
```

---

### For Consultancy Engagements

Structure as engagement stages:

```markdown
#### Stage [N]: [Stage Name] — [Duration]

**Objective:** [What we will achieve in this stage]

**Methods:**
- [Workshop / Interview / Analysis / Review]

**Outputs:**
- [Specific document or recommendation delivered]

**Participants Required from Client:**
- [Role 1] — [Approximate time commitment]
- [Role 2] — [Approximate time commitment]
```

---

Always include an explicit in/out of scope summary:

---

## Change Control

Add this standard paragraph at the end:

> Any changes to the scope described in this document will be managed through a formal Change Request (CR) process. Changes may affect cost, timeline, or resource allocation. No change will be implemented without written agreement from both parties.
